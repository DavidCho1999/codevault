================================================================================
[1] Slash Command란?
================================================================================

정의
----
- 자주 쓰는 프롬프트를 저장한 "단축키"
- 마크다운 파일 하나 = 명령어 하나
- /명령어 입력하면 실행

예시
----
commit.md 파일 만들면 -> /commit 으로 실행


================================================================================
[2] 저장 위치
================================================================================

개인용 (모든 프로젝트에서 사용)
-------------------------------
~/.claude/commands/
-> /help에서 (user)로 표시

프로젝트용 (이 프로젝트만)
--------------------------
.claude/commands/
-> git 커밋하면 팀원과 공유됨
-> /help에서 (project)로 표시

================================================================================
[4] 고급 설정 (Frontmatter)
================================================================================

파일: .claude/commands/commit.md

---
description: Git 커밋 생성
argument-hint: [message]
allowed-tools: Bash(git add:*), Bash(git status:*), Bash(git commit:*)
model: claude-3-5-haiku-20241022
---
다음 메시지로 git commit 생성: $ARGUMENTS


옵션 설명
---------
description: 설명 (/help에 표시)
argument-hint: 인자 힌트
allowed-tools: 사용 가능 도구 제한
model: 사용할 모델 (haiku = 빠름, opus = 정확)

[6] BORIS CHERNY의 Slash Commands
================================================================================
URL: https://paddo.dev/blog/how-boris-uses-claude-code/

[핵심 철학]
"반복하는 프롬프트는 모두 Slash Command로"
"하루에 수십 번 쓰는 워크플로우를 자동화"

[실제 사용하는 Commands]

1. /commit-push-pr (가장 많이 사용)
-----------------------------------
용도: 커밋 -> 푸시 -> PR 한 번에
빈도: 하루 수십 번
특징: inline bash로 git status 미리 계산

예상 내용:
---
description: 커밋, 푸시, PR 생성
allowed-tools: Bash(git:*)
---
!git status
!git diff --staged

위 변경사항을 분석하고:
1. 적절한 커밋 메시지 생성
2. 커밋 실행
3. 원격에 푸시
4. PR 생성


2. /feature-dev (기능 개발)
---------------------------
용도: 기능 개발 전체 워크플로우
단계:
1. 뭘 만들지 물어봄
2. 스펙 작성
3. 상세 계획
4. TODO 리스트
5. 단계별 실행


3. /code-review (코드 리뷰)
---------------------------
용도: PR 첫 패스 리뷰
특징: Claude가 먼저 리뷰, 사람이 최종 승인

[7] 커뮤니티 인기 Commands
================================================================================
URL: https://github.com/wshobson/commands

Git 관련
--------
/commit-fast    - 자동 커밋 메시지 생성 후 바로 커밋
/create-pr      - 브랜치 생성 -> 커밋 -> PR 제출
/fix-pipeline   - CI 실패 로그 분석 -> 수정

코드 품질
---------
/review-changes - 변경사항 코드 리뷰
/test-and-fix   - 테스트 실행 -> 실패하면 수정
/refactor       - 선택한 코드 리팩토링

분석
----
/explain        - 코드 설명
/first-principles - 문제 근본 분석


================================================================================
[8] 실전 예시: 자주 쓰는 Commands
================================================================================

예시 1: 빠른 커밋 (/quick-commit)
---------------------------------
파일: .claude/commands/quick-commit.md

---
description: 빠른 git 커밋
model: claude-3-5-haiku-20241022
---
!git status
!git diff --staged

위 변경사항으로 간결한 커밋 메시지 생성하고 커밋해줘.
Conventional Commits 형식 사용.


예시 2: GitHub 이슈 수정 (/fix-issue)
-------------------------------------
파일: .claude/commands/fix-issue.md

---
description: GitHub 이슈 수정
argument-hint: [issue-number]
---
GitHub 이슈 #$ARGUMENTS 를 분석하고 수정해줘.

1. 이슈 내용 파악
2. 관련 코드 찾기
3. 수정 방안 제시
4. 코드 수정
5. 테스트


예시 3: 코드 설명 (/explain-code)
---------------------------------
파일: .claude/commands/explain-code.md

---
description: 코드를 비전공자도 이해하게 설명
---
$ARGUMENTS

위 코드를 다음 규칙으로 설명해줘:
- 전문 용어 피하기
- 비유 사용하기
- 단계별로 설명

[9] 파워 유저 팁
================================================================================

Boris Cherny
------------
- inline bash (!)로 git status 미리 계산 -> 빠름
- .claude/commands/에 저장 -> git으로 팀 공유
- 하루 수십 번 쓰는 것만 Command로

Anthropic 공식
--------------
- "두 번 하면 자동화하라"
- model: haiku 쓰면 빠름 (간단한 작업)
- 복잡한 작업은 opus

[10] Skill vs Slash Command
================================================================================

+------------------+------------------+------------------+
| 구분             | Slash Command    | Skill            |
+------------------+------------------+------------------+
| 호출             | 수동 (/명령어)   | 자동 (Claude)    |
| 파일             | 단일 .md 파일    | 폴더 + SKILL.md  |
| 용도             | 빠른 반복 작업   | 복잡한 워크플로우 |
| 스크립트         | inline bash만    | 별도 파일 가능   |
+------------------+------------------+------------------+

선택 기준
---------
- 내가 직접 실행 -> Slash Command
- Claude가 알아서 -> Skill
