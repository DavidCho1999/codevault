================================================================================
[1] Skill이란?
================================================================================

정의
----
- Claude에게 특정 작업을 가르치는 "훈련 매뉴얼"
- 한 번 설명하면, 다시 설명할 필요 없음
- 폴더 + SKILL.md 파일로 구성

핵심 특징
---------
- 자동 발동: Claude가 알아서 필요할 때 사용
- Slash Command와 다름: /command 입력 필요 없음
- 스크립트 포함 가능: Python, Bash 등 실행 가능


================================================================================
[2] Skill vs Slash Command vs Agent 비교
================================================================================

+------------------+------------------+----------------------------------+
| 기능             | 호출 방식        | 용도                             |
+------------------+------------------+----------------------------------+
| Slash Command    | 수동 (/command)  | 빠른 반복 작업                   |
| Skill            | 자동 (Claude)    | 복잡한 워크플로우 + 파일 포함    |
| Agent            | 자동/수동        | 병렬 실행, 독립 컨텍스트         |
+------------------+------------------+----------------------------------+

언제 뭘 쓰나?
-------------
- 명시적 제어 필요 -> Slash Command
- Claude가 알아서 적용 -> Skill
- 격리된 실행 필요 -> Agent

[4] Skill 만들기 - 실전 예시
================================================================================

예시 1: 커밋 메시지 생성기
--------------------------
폴더: .claude/skills/commit-message/

SKILL.md:
---
name: commit-message
description: Git 커밋 메시지 생성. staged 변경사항 분석.
---
# 커밋 메시지 생성

## 지침
1. git diff --staged 실행
2. 변경사항 분석
3. 커밋 메시지 제안:
   - 제목: 50자 이하
   - 본문: 상세 설명
   - 영향받는 컴포넌트 나열


예시 2: 코드 리뷰어
-------------------
폴더: .claude/skills/code-reviewer/

SKILL.md:
---
name: code-reviewer
description: 코드 리뷰 수행. PR, 변경사항 검토 시 사용.
allowed-tools: Read, Grep, Glob
---
# 코드 리뷰

## 체크리스트
- [ ] 보안 취약점
- [ ] 성능 이슈
- [ ] 코드 스타일
- [ ] 테스트 커버리지

1. 이번 주 완료

3. 블로커/이슈

[5] 고급 설정
================================================================================

도구 제한 (allowed-tools)
-------------------------
---
name: safe-reader
description: 파일 읽기 전용
allowed-tools: Read, Grep, Glob
---
-> 이 스킬 활성화 시 Read, Grep, Glob만 사용 가능


모델 지정 (model)
-----------------
---
name: complex-analysis
description: 복잡한 분석 작업
model: claude-sonnet-4-20250514
---


수동 호출만 허용 (disable-model-invocation)
-------------------------------------------
---
name: dangerous-operation
description: 위험한 작업
disable-model-invocation: true
---
-> Claude가 자동 호출 불가, /dangerous-operation으로만 실행

[8] Best Practices
================================================================================

1. SKILL.md 500줄 이하로
   - 길면 별도 REFERENCE.md 파일로 분리
   - Claude가 필요할 때만 참조

2. description을 잘 쓰기
   - Claude가 description으로 언제 쓸지 판단
   - 키워드 풍부하게

3. 점진적 공개 (Progressive Disclosure)
   - 핵심만 SKILL.md에
   - 상세 내용은 별도 파일
   - 컨텍스트 절약

4. 신뢰할 수 있는 소스만
   - Skill은 코드 실행 가능
   - npm 패키지처럼 취급

[12] 파워 유저들의 Skill/Slash Command 활용법 (추가)

BORIS CHERNY (Claude Code 창시자)
---------------------------------
URL: https://paddo.dev/blog/how-boris-uses-claude-code/

[핵심 철학]
"반복하는 프롬프트는 모두 Slash Command로 만든다"
-> 본인도 편하고, Claude도 호출 가능

[실제 사용하는 Slash Commands]
1. /commit-push-pr - 하루 수십 번 사용 (가장 많이)
   - 코드 저장 -> 커밋 -> 푸시 -> PR 자동화
   - inline bash로 git status 미리 계산

2. /feature-dev - 기능 개발 워크플로우
   - "뭘 만들지 물어봐 -> 스펙 작성 -> 계획 -> TODO -> 단계별 실행"

3. /code-review - 모든 PR 첫 패스
   - Claude가 먼저 리뷰, 사람이 최종 승인

[저장 위치]
.claude/commands/ 에 저장 -> git 커밋 -> 팀 공유

[놀라운 점]
"내 셋업은 의외로 바닐라다. Claude Code는 기본 상태로도 잘 작동한다."


SIMON WILLISON (Django 공동 창시자)
-----------------------------------
URL: https://simonwillison.net/2025/Oct/16/claude-skills/

[핵심 주장]
"Skills가 MCP보다 더 큰 딜일 수도 있다"

[이유]
1. 단순함: MCP는 복잡한 프로토콜, Skills는 마크다운+YAML
2. LLM 철학: "텍스트 던지고 모델이 알아서" = LLM 본질
3. CLI 대체: MCP 대신 CLI로 충분, LLM은 --help 읽을 줄 앎
4. 공유 쉬움: 파일 하나면 됨

================================================================================
[요약] 파워 유저들의 공통점
================================================================================

1. 복잡하게 안 씀 (바닐라 선호)
2. 반복 작업만 Slash Command로
3. Skills > MCP (단순함 선호)
4. 복잡한 커스텀 = 안티패턴
5. "뭘 입력해도 작동해야 진짜"
